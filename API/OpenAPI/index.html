Soupify™

This is a backend server API for storing recipes and meals we coocked. RestfulAPI for the re-branding project of Engineering-Students-Recipes Company to their new company, Soupify™. This project was creating as a group project for Web-Development-Environments course @Ben-Gurion University. For more details on the process/design please read the linked README.md file.

Authorization: A logged-in user must update/delete his resources ONLY! Otherwise, he will receive 403 Unauthorized Error.

Base URL: {your_custom_url}/api

Validity: All parameters sent to any API endpoint are validated according to the following criteria:

  • Required vs Optional
  • Numeric vs Alpha vs Alphanumeric
  • Unique vs Multiple
  • Min/max length of strings
  • Min/max values of numbers
  • Date formats

Request Headers: All request headers must have the following fields:

Authorization: Bearer <user's auth JSON Web Token>

You can use this header token for admin access:

Pagination: Pages and Limits are query parameters which are designed for paginate freely. Pages and Limits must be NUMERIC.

  • page is the page number jump to.
  • Limit is the number of items per page.
  • offest calculate as (page - 1) x limit
  "page": 1, 2, 3 ...
  "limit": 5, 10, 15 ...

Response Format: Responses have a specific format. We will use the User model to depict an example, but User can be any other model.

  • Successful Response with an array of users:
 {
  "Pagination": {
    "page": 1,
    "per_page": 5,
    "pre_page": null,
    "next_page": 2,
    "total": 100,
    "total_pages": 20
    },
    "data": [
      {
        * user object *
      }
  ]
}
